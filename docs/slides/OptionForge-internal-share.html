Agent-First Deck

OptionForge 内部分享

这套脚手架已经从“统一命令面”收口为“模板仓库 + focus 资产 + verification profile”。我们不再强调安装式入口，而是强调 agent 如何稳定地理解意图、边界和验证证据。

当前定位

  • 模板仓库优先，直接在现有代码上迭代策略。
  • strategy_spec.toml 负责表达意图。
  • .focus/context.json 和配套 markdown 负责表达当前可编辑面。
  • 验证以 focus.smoke 和 focus.full 为主，而不是靠命令包装器组织叙事。

核心资产

  • focus/strategies/*/strategy.manifest.toml 描述 workflow、pack、surface。
  • .focus/WORKFLOWS.md 统一说明 runtime、backtest、monitor 与 verification profile。
  • .focus/TASK_ROUTER.md 提供 pack 级阅读和验证路径。
  • tests/TEST.md 沉淀当前策略的验证摘要。

团队收益

  • 减少脚手架使用门槛，直接面向代码编辑。
  • 把“怎么改”交给 manifest 与 generated assets，而不是 help 文本。
  • 让 focus 宽度、测试 selectors 和关键产物对人和 agent 都可见。

后续维护原则

  • 变更 workflow 语义时，优先改 source layer，再刷新生成资产。
  • 避免把 pack 逻辑重新推回广义入口层。
  • 只有确实需要执行证据时，才扩展到 runtime 或 backtest 工作流。
